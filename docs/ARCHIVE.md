# Archive and Experiment History

This repository preserves a large amount of research history in addition to its curated documentation and validation paths.

## Purpose

Some files and naming patterns in this repo exist to preserve the chronology of the work rather than to present a polished public API.

Examples include:

- `onebit/research/novel_ideas_v*.py`
- corrected follow-ups such as `*_corrected.py`
- root-level result logs such as `results_v*.txt`
- one-off benchmark, diagnostic, and comparison scripts

These files are intentionally retained because they document:

- failed ideas,
- partial successes,
- revisions after stricter testing,
- and the actual path by which the project’s conclusions evolved.

## How to read the repo

If you are new to the repository, start with the curated materials:

1. `README.md`
2. `RESEARCH.md`
3. `REPRODUCIBILITY.md`
4. `docs/HONEST_ASSESSMENT.md`
5. `docs/PROJECT_ANALYSIS_SUMMARY.md`
6. `docs/REPOSITORY_GUIDE.md`

Then use the archived experiment files as historical context, not as the primary source of the repo’s current validated claims.

## Naming convention note

Names like `novel_ideas_v7.py` or `novel_ideas_v24_corrected.py` should be interpreted as:

- chronological research checkpoints,
- not stable package interfaces,
- not necessarily the strongest current method,
- and not the preferred entry points for reproduction.

## Public interpretation

The presence of these archived files means this repository should be understood as a **serious research workspace with preserved history**, not a minimal polished benchmark package.

That is intentional.
